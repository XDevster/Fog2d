           FOG2D ENGINE: ТЕХНИЧЕСКИЙ СПРАВОЧНИК

В этом файле разобраны все "внутренности" твоего движка.

------------------------------------------------------------
1. КЛАСС GameObject (Твой кирпичик)
------------------------------------------------------------
Зачем: Это шаблон для любого объекта в игре (игрок, враг, пуля).

- __init__(self, x, y, char, color, gravity, draggable):
    * x, y: Координаты. Всегда передавай числа (int или float).
    * char: Символ, который мы видим (например, 'P').
    * color: Цвет из Colorama (например, Fore.RED).
    * gravity (bool): Если True, объект постоянно падает вниз.
    * draggable (bool): Флаг для перетаскивания (задел на будущее).

- vx, vy: Твои главные рычаги.
    * vx: Скорость по горизонтали. Положительная = вправо, Отрицательная = влево.
    * vy: Скорость по вертикали. Отрицательная = ПРЫЖОК вверх.

------------------------------------------------------------
2. КЛАСС Scene (Мир игры)
------------------------------------------------------------
Зачем: Хранит карту и логику конкретного уровня.

- __init__(self, raw_map): 
    * Автоматически выравнивает карту (Padding), чтобы строки 
      были одной длины. Это защищает от IndexError.
- update(self, engine, dt):
    * Здесь ты пишешь правила игры. engine дает доступ к игроку, 
      а dt помогает сделать движение плавным (независимым от FPS).
- render_ui(self, engine):
    * Используй print() здесь. Движок сам поднимет курсор в 
      верхнюю часть экрана (строки 1-4).

------------------------------------------------------------
3. КЛАСС Fog2D (Ядро / Движок)
------------------------------------------------------------
Зачем: Управляет циклом игры, физикой и отрисовкой.

- switch_scene(scene):
    * Очищает экран и загружает новую карту. Используй это для 
      перехода между уровнями (например, из леса в пещеру).
- get_tile(x, y):
    * Самая важная функция для коллизий. Она возвращает символ 
      карты в этой точке. 
      Пример: if engine.get_tile(p.x, p.y) == "#": стоим у стены.
- _draw(x, y, char):
    * Магия Colorama. Прыгает в точку (x, y) и рисует символ. 
      Добавляет +5 к Y, чтобы оставить место под UI сверху.
- run(fps):
    * Бесконечный цикл. Внутри него: Физика -> Коллизии -> Отрисовка.

------------------------------------------------------------
4. ПОЧЕМУ ТАК ВАЖНЫ БИБЛИОТЕКИ?
------------------------------------------------------------
- colorama: Без нее консоль умеет только печатать строку за строкой. 
  Colorama позволяет движку перерисовывать только нужные символы, 
  создавая иллюзию анимации.
- keyboard: В стандартном Python нажатие клавиши блокирует программу. 
  Эта библиотека позволяет "слушать" кнопки в фоновом режиме, 
  пока игрок летит или падает.
